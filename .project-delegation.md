# NFL Dashboard — Delegation Contract

> Specifies what can be safely delegated to GitHub Actions and other executors.

## What Can Run on GitHub Actions (Async)

- ✅ **Tests**: `npm run test` (emit JSON report to `test-results.json`)
- ✅ **Lint**: `npm run lint:fix` (auto-fix mode, emit diff to receipt)
- ✅ **Database migrations**: `npx supabase db push` (emit migration receipt)
- ✅ **GHA workflows**: Scheduled tasks already configured in `.github/workflows/`
- ✅ **Type check**: `npx tsc --noEmit` (emit type errors to receipt)
- ❌ **Dev server**: `npm run dev` (interactive, requires browser)
- ❌ **Vitest watch**: Watch mode requires UI interaction
- ❌ **Manual code review**: Requires human judgment

## Delegation Format

```json
{
  "project": "nfl_dashboard",
  "task_id": "F-14-supabase-migration",
  "delegated_at": "2026-05-20T10:45:00Z",
  "executor": "GHA|M6-manual",
  "expected_duration_minutes": 5,
  "command": "npx supabase db push",
  "receipt_path": ".nfl/receipts/migration-F14-2026-05-20.json",
  "resume_condition": "receipt.status == 'success'",
  "fallback": "manual apply in Supabase dashboard"
}
```

## GHA Executor Configuration

Scheduled workflows in `.github/workflows/`:

- `test.yml` — runs on PR and schedule (daily 9 AM)
- `lint.yml` — runs on PR and schedule (daily 8 AM)
- `db-migrate.yml` — manual trigger (requires branch with migration files)

## Resume at Home

1. **Check GHA status**: GitHub Actions tab in repo
2. **Read receipts**: `cat .nfl/receipts/*.json | jq '.status'`
3. **If migration failed**: apply manually in Supabase dashboard
4. **If tests failed**: check `test-results.json` and fix locally
5. **Commit fixes**: `git add . && git commit -m "fix: [issue]" && git push origin main`
